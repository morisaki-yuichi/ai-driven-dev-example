# 実験02 — CLAUDE.md 無しで既存リポジトリを触らせる（規約無視）

> 第2部の実験。形式は **予想 → 観察 → 検出網（＝CLAUDE.md）**。
> 記録日 2026-07-05 / claude-opus-4-8。
> 関連: [`docs/concepts/02-context-and-spec.md`](../../docs/concepts/02-context-and-spec.md)・[`CLAUDE.md`](../../CLAUDE.md)。

## ねらい

検証網（第1部）は「正しさ」を守るが、「このプロジェクトの流儀」は守らない。
CLAUDE.md が**無いと** AI が規約をどう外し、**有ると**何が変わるかを対比する。

## 予想

「概念解説を1本追加して」とだけ頼むと、CLAUDE.md が無い AI は一般論で動き、
次を外す：言語（日本語のはずが英語）・置き場所（`docs/concepts/NN-*.md` のはずがルート）・
形式（3点形式を無視）・手順（ブランチ／セッションログ無し）。**テストは通っても流儀は守られない。**

## 観察（代表例の対比）

CLAUDE.md 無し（規約を外した一次出力の再構成）:

    # Concept: Context Window
    The context window is the amount of text a model can attend to at once...
    （置き場所: リポジトリ直下 notes.md / 英語 / 3点形式なし / session-log もブランチも無し）

CLAUDE.md 有り（整備後に期待される形）:

    docs/concepts/03-context-window.md
    # 概念解説 03 — コンテキスト窓
    - ① 一言定義 …  ② なぜ必要か・事故例 …  ③ このリポジトリでの実例 …
    （＋フィーチャーブランチ / セッションログ / PR → CI 緑 → マージ）

## 検出網（＝CLAUDE.md が埋める穴）

| 違反の種類 | 検証網(ruff/mypy/pytest)で捕まる？ | CLAUDE.md で防げる？ |
|-----------|:---:|:---:|
| 型・構文の誤り | ○ | － |
| 置き場所・命名 | ✗ | ○ |
| 言語（日本語/英語） | ✗ | ○ |
| 3点形式・体裁 | ✗ | ○ |
| ワークフロー（ブランチ/ログ/PR） | ✗ | ○ |

→ **テストで表現できない規約は、CLAUDE.md が受け持つ**。検証網と CLAUDE.md は補完関係。
「動く」と「流儀に合う」は別問題。

## 教訓

1. 検証網が緑でも、規約・体裁・手順は守られない。CLAUDE.md がその穴を埋める。
2. CLAUDE.md は **“常時文脈”＝毎回読ませる**ことに価値がある。会話で一度言うだけでは、
   セッションの圧縮で消える（第2部後半のコンテキスト寿命管理に続く）。

## 正直さについて

上の2出力は代表的な**再構成**（非決定性のため、実際の逸脱は毎回違う）。
再現できるのは「CLAUDE.md の有無で観点表の ✗ が ○ に変わる」という**構造**のほうであり、
この教材が固定したいのはその構造である。
