# 実験03 — コンテキスト圧縮と記憶の外部化

> 第2部の実験。形式は **予想 → 観察 → 検出網**。記録日 2026-07-05 / claude-opus-4-8。
> 関連: [`concepts/03`](../../docs/concepts/03-context-lifetime.md)・[`concepts/00`](../../docs/concepts/00-llm-nature.md)（コンテキスト窓）。

## ねらい

窓には上限があり、超えると圧縮で序盤が落ちる。決定を「会話」に置くと消え、
「ファイル」に置くと残る——この対比を確かめる。

## 予想

長いセッションを圧縮まで走らせると、序盤に**会話で**決めた制約は曖昧化・欠落しうる。
一方、序盤の決定を**ファイルに外部化**しておけば、何ターン後でも `grep` 一発で取り出せる。

## 観察

### ファイル側（実行・本物の出力）

本セッションは序盤（session 000）の決定をファイルへ外部化した。今それを取り出す:

```console
$ grep -rn "編集版のみ\|5部\|サブディレクトリ" docs/session-logs/000-setup.md
000-setup.md:20: … 本リポジトリのサブディレクトリに取り込み（monorepo 的）…
000-setup.md:21: … 5部＝5スプリント、1本 3〜4h 目安 …
000-setup.md:22: … 編集版のみ（要点＋採用/棄却の判断理由）…

$ ls docs/session-logs/
000-setup.md 001-sprint0.md 002-… 003-… 004-… 005-… 006-…
```

→ 何十ターン経っても、序盤の決定はファイルから**完全な形**で戻る。

### 会話側（既知の挙動・再構成）

同じ制約を「会話でだけ」言っていた場合、圧縮後に問い直すと
「たしか編集版…だっけ？」のように曖昧化・欠落する。ここは長文脈・非決定性のため
その場で確実には再現できず、既知の失敗モードとして示す（下記「正直さについて」）。

## 検出網（＝記憶の外部化）

| 置き場所 | 圧縮後 | 取り出し |
|----------|:---:|----------|
| 会話（揮発） | 失われうる | 不可（消えたら終わり） |
| ファイル（永続） | 残る | `grep`／明示読込でいつでも |

→ **決定はファイルへ**。加えて「圧縮時に残す情報」を指示するのも補助になる。

## 教訓

1. 圧縮は起きる前提。防ぐのでなく「落ちても困らない」設計にする。
2. 会話で合意しただけの制約は、書かなければ無かったことになる。
3. この教材の 000〜007 ログ自体が、外部化の実証データ。

## 正直さについて

「ファイルは残る」は実行で確認した本物。「会話は消える」側は、長セッションを圧縮まで
走らせる必要があり本ドリルでは確実に再現できないため、既知の挙動として提示する。
固定できるのは「外部化した決定が生き残る」という構造のほうで、それがこの実験の主眼。
