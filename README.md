# ai-driven-dev-example — AI駆動開発の実践教材

姉妹作 **todo-app-example2**（Laravel・手書き）/ **todo-app-example3**（FastAPI+React・手書き）の
第3周。**同じ TODO アプリを AI 駆動で作り直し**、人間が書いた場合との差分を比較できる教材です。

## 統括仕様

- [`docs/initial-prompt.md`](docs/initial-prompt.md) — 役割・設計原則・モデル/コスト方針・進め方。
  全スプリントはこのプロンプトを起点とする。

## 構成（5部＝5スプリント）

| | 部 | 主題 |
|---|----|------|
| 第0部 | 概念 | LLM の性質と「なぜ検証なしで信じてはいけないか」 |
| 第1部 | 檻と土台 | DevContainer/sandbox・権限設計・検証網（テスト/lint/CI） |
| 第2部 | 文脈と仕様 | CLAUDE.md・仕様駆動・レビュー技術・コンテキストの寿命管理 |
| 第3部 | 拡張 | MCP・skills・sub-agents |
| 第4部 | 運用 | 既存コードベースへの追加開発の自律ループ |

## リポジトリの歩き方

- `docs/concepts/` — 概念解説集（①一言定義 ②なぜ必要か・事故例 ③このリポジトリでの実例）
- `docs/session-logs/` — 編集版セッションログ（要点＋採用/棄却の判断理由。判断が読める粒度）
- `experiments/` — わざと失敗を見る実験（**予想 → 観察 → 検出網** の3点セット）

## 追体験の単位

同じプロンプトでも同じコードは出ない（非決定性）。だから追体験は「コードの一致」ではなく
**「指示 → 出力 → 評価 → 修正指示」＋判断の注釈**で行い、
**合否は受け入れ条件とテストで判定**する。
