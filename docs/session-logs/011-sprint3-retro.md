## 実データ（Explore・各1回）

| 方式 | 時間 | トークン |
|------|-----:|---------:|
| 単一（5本） | 25.9s | 16,446 |
| 分割A（00-02） | 27.0s | 13,459 |
| 分割B（03-04） | 18.2s | 10,466 |
| **分割 計/並列** | **≈27.0s** | **23,925**（＋統合分） |

→ 分割は**時間短縮ゼロ（むしろ微増）× コスト約1.45倍**、加えて shard 間の体裁ずれを親が統合。
**「並列だから得」は幻想。分割は設計判断。**
