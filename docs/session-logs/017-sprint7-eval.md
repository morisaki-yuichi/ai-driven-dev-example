## 判断（採用/棄却）

| 判断点 | 採用 | 理由 | 棄却 |
|--------|------|------|------|
| 採点方式 | **決定スコア**（述語で採点） | 再現可能・採点バイアスが乗らない | LLM-as-judge（採点コスト/バイアスは別テーマ→将来 Try） |
| モデル出力 | 収録フィクスチャ＋live 対応ランナー | API 不在。実スコアは本物、出力は明記した収録 | 実行の捏造（禁止）／API 前提で書くだけ（動かせない） |
| モデルID/価格 | skill で実測固定＋Models API 注記 | 知識の鮮度（推測で書かない） | 記憶から価格を書く（陳腐化・誤り） |
| Solver 抽象 | `Case→str` で recorded/live を差し替え | 採点と出力取得を分離（live 化が1行） | live をランナーに直書き（差し替え不能） |
| 題材 | make_slug（期待特性が明快） | 決定述語に落としやすい | エージェント挙動（採点特性が曖昧・今回スコープ外） |
| 置き場 | `eval/` を新設しゲートに追加 | 「検証網に載せる」を満たす・概念が独立 | tools に同居（観測と評価が混ざる） |

## 検収観点

- 見た目が良い B を、eval が**数字の悪化**として捕まえているか（A=1.000→B=0.571）。

- 実スコア（計算）と収録出力（見立て）の区別が明記されているか（正直さ）。
