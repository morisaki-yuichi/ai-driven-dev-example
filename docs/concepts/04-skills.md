# 概念解説 04 — skills（必要時読込の記憶層）

> 第3部。3点形式。執筆日 2026-07-05 / claude-opus-4-8。前提: 概念03「4層の記憶モデル」。

MCP が「外部ツールへの接続」なら、skill は **手順の記憶**。CLAUDE.md（常時）と
会話（揮発）の中間にある「**必要な時だけ読み込む**」層。

## 1. 必要時読込（skill）

- **① 一言定義**: 特定タスクの手順を、関連する時だけ読み込む記憶。
- **② なぜ必要か・事故例**: 手順を全部 CLAUDE.md に書くと肥大化し、常時文脈のノイズになる
  （長すぎて読まれない・窓を圧迫）。必要時だけ展開すれば土台を軽く保てる。
- **③ このリポジトリでの実例**: 何度も繰り返した「出荷手順」を
  [`.claude/skills/ship-change/`](../../.claude/skills/ship-change/SKILL.md) に切り出した。

## 2. いつ skill にするか（CLAUDE.md との線引き）

- **① 一言定義**: 「**常に効く規約**」は CLAUDE.md、「**特定タスクの手順**」は skill。
- **② なぜ必要か・事故例**: 全部 CLAUDE.md だと長くて読まれず窓を食う。全部会話だと圧縮で消える。
- **③ このリポジトリでの実例**: 規約（言語・置き場所・ワークフローの原則）は CLAUDE.md、
  出荷の手順詳細は `ship-change` skill。CLAUDE.md からは skill を**軽く指す**だけにした。

## 3. skill の形

- **① 一言定義**: `.claude/skills/<name>/SKILL.md`。frontmatter の `description` が「**いつ使うか**」で、
  モデルはそれを見て必要時に選ぶ。
- **② なぜ必要か・事故例**: `description` が的確でないと、呼ばれない／呼ばれ過ぎる。
- **③ このリポジトリでの実例**: `ship-change` の `description` に「変更を出荷するとき」と明示した。

---

## まとめ

4層（**常時＝CLAUDE.md / 必要時＝skills / 揮発＝会話 / 永続＝ファイル**）を使い分ける。
skills は「手順を必要時に配る」層。次は **sub-agents**——手順ではなく**作業そのものを委譲**する層へ。
