# 概念解説 06 — hooks（確率的な指示 → 決定的な足場）

> 第5部。3点形式。執筆日 2026-07-05 / claude-opus-4-8。前提: 概念01「檻と権限」/ 概念04「skills」。

CLAUDE.md も skill も、モデルへの**お願い**にすぎない。読むかどうか・従うかどうかは確率的で、
長い会話・圧縮・疲れた文脈で破られる。hook は、その保証を**ハーネス側の機械**に移す層。
「指示して祈る」から「経路を塞ぐ」へ。

## 1. hook（決定的なガードレール）

- **① 一言定義**: ツール実行の前後などライフサイクルの決まった点で、ハーネスが**必ず**走らせる
  コマンド。標準エラーに理由を出し **exit 2 で実行をブロック**、exit 0 で許可（Claude Code の規約）。
- **② なぜ必要か・事故例**: 「main に直コミットしない」を CLAUDE.md に書いても、それは確率的な指示。
  実際、[`session-logs/013`](../session-logs/013-sprint4-priority.md) の直後に **main 直コミット事故**が起きた。
  PreToolUse hook にすれば、同じ操作は**モデルの判断を介さず**通らない経路になる。
- **③ このリポジトリでの実例**: [`.claude/hooks/guard.py`](../../.claude/hooks/guard.py) が
  「main 直コミット/push・秘密ファイル書き込み・広域 `rm -rf`」を exit 2 で止める。
  同じ `git commit` でも**ブランチ名という機械的事実**だけで block/allow が決まる（[実験06](../../experiments/06-hooks/)）。

## 2. ライフサイクルのどこで介入するか（Pre/Post/Stop）

- **① 一言定義**: `PreToolUse`＝ツール実行**前**（唯一ブロックできる）、`PostToolUse`＝**後**（記録・整形）、
  `Stop`＝応答終了時（取りこぼしの最終確認）。settings.json の `hooks.<イベント>[].{matcher, hooks}`。
- **② なぜ必要か・事故例**: 「危険を止める」のは実行前でしか間に合わない。「検証を忘れた」の検出は
  停止時が最後の砦。役割の違う点に別々の hook を置く。
- **③ このリポジトリでの実例**: Pre=ガード（[`guard.py`](../../.claude/hooks/guard.py)）、
  Post(Edit|Write)=`ruff --fix` で自動整形（[`ruff_fix.py`](../../.claude/hooks/ruff_fix.py)）、
  Post(全ツール)=JSONL トレース（[`trace.py`](../../.claude/hooks/trace.py)・第6部の入力）、
  Stop=`pytest -q` を走らせ**赤なら停止をブロック**（検証の取りこぼし防止）。

## 3. 決定的ゆえのコスト（誤爆・fail-open・檻との関係）

- **① 一言定義**: 機械的な規則は**取りこぼし（false negative）と誤爆（false positive）**の綱引き。
  ガードは安全側（誤爆寄り）に倒しつつ、開発を止めすぎない設計（fail-open）が要る。
- **② なぜ必要か・事故例**: 文字列一致は「言及しただけ」でも過剰反応する。実験06では、payload に
  `rm -rf` と**書いただけ**の Bash コマンドが（実行していないのに）ブロックされた。想定外入力で
  ガードが暴発すると作業が止まる。だから guard は列挙した危険だけ閉じ、解釈不能は素通り（fail-open）にした。
- **③ このリポジトリでの実例**: hook は**決定的な足場**であって最終防壁ではない。真の封じ込めは
  第1部の**檻（DevContainer）**（[実験01](../../experiments/01-blast-radius/)）。hook はその手前で、
  正しさ（検証網）と被害範囲（檻）の**間**に「決定的な不変条件」を足す層。

---

## まとめ

記憶の4層（常時=CLAUDE.md / 必要時=skills / 揮発=会話 / 永続=ファイル）は、どれも**モデルが読む**確率的な層。
hook は初めて**モデルを介さず**効く層＝決定的な足場だ。保証したい不変条件は、指示ではなく hook に落とす。
ただし「守る hook」自身も検証網に載せて守る（[`test_hooks.py`](../../experiments/06-hooks/test_hooks.py)）。
次は第6部——この hook が吐く JSONL トレースを**読んで集計**し、可観測性の土台にする。
