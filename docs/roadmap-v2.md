## 第5部 hooks — 決定的な足場

- **ゴール**: 「skill/CLAUDE.md（確率的）→ hook（決定的）」を体験し、保証をハーネス側に移す。
- **失敗実験**: hook 無しで「main 直コミット／秘密書き込み／検証忘れ」が通る →
  PreToolUse/PostToolUse hook が機械的に防ぐ。予想→観察→検出網(=hook)。
  - 生きた素材: [`session-logs/013`](session-logs/013-sprint4-priority.md) 直後に実際起きた「main 直コミット」事故。
- **実装（`.claude/settings.json`）**:
  - PostToolUse(Edit|Write) → `ruff check --fix`（検証網の自動化）
  - PreToolUse → 秘密ファイル／`main` への危険操作をブロック（決定的ガードレール）
  - Stop → `pytest -q`（検証の取りこぼし防止）
  - PostToolUse(全ツール) → JSONL でツール呼び出しをログ（第6部の入力）
- **DoD**: hook 群 ＋ ブロック実演（本物の出力）＋ 概念06 ＋ セッションログ。
- **依存**: なし（最初）。正確な設定スキーマは公式ドキュメント／`update-config` skill で確認（知識の鮮度）。

## 第6部 可観測性 — 後から再構成できる

- **ゴール**: 行動・コスト・時間を計上し、最適化とデバッグの土台にする。
- **実装**: 第5部の JSONL トレースを読む集計（`tools/trace_summary.py`。我々の検証網＝mypy/pytest に載せる）。
  ツール呼び出し数・所要・（取れれば）トークン。sub-agent usage 計測を発展。
- **失敗/比較実験**: トレース有無で「なぜ遅い/失敗したか」を診断できるか比較。観測できないと最適化できない。
- **DoD**: hook → JSONL → 集計の一本道 ＋ 実トレースの読み ＋ 概念07 ＋ ログ。
- **依存**: 第5部のログ hook。

## 第7部 評価（eval）— 良くなったかを測る

- **ゴール**: プロンプト/ツール/ハーネス変更の合否を**質**で判定（コードの tests ではなく）。
- **スコープ（決定）**: 最小＝**決定スコア中心**（期待特性をコードで採点）。
- **実装**: 小さな eval セット（タスク＋期待特性）＋ ランナー（Claude API / Agent SDK で実行 → 採点）。
  題材は `make_slug` やエージェント挙動。
- **失敗実験**: プロンプトを「改善」→ eval が**実は悪化**を検出（vibes は気づかない）。予想→観察→検出網(=eval)。
- **DoD**: eval セット ＋ ランナー ＋ 実スコア ＋ 概念08 ＋ ログ。
- **依存**: 第6部のコスト/時間計上。Claude API 利用（コスト発生）。exact モデルID／価格は着手時に公式で引く。
- **将来 Try（今回スコープ外）**: LLM-as-judge ／ モデル階級比較（Haiku/Sonnet/Opus の品質×時間×コスト）。
